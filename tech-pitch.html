TECH CARD / LAB NOTE

Pitchカード

作ったものを、伝わる物語にする

このカードの出番

  • ●企業の方に発表する
  • ●短時間で魅力を伝えたい
  • ●分析とUIを1つのストーリーにまとめたい

作れるもの

3〜5分発表構成スライドデモ画面質問したいこと

ざっくり言うと

ピッチは、ただの発表ではありません。誰のどんな困りごとに気づき、どんなデータを使い、どんなプロダクトにしたのかを短く伝える技術です。

合言葉

「作ったものを、伝わる物語にする」

図で見る考え方

flowchart LR A[問題] --> B[アイデア] B --> C[データ] C --> D[分析] D --> E[プロトタイプ] E --> F[次にやりたいこと]

HOW TO USE

使い方はこの順番

1

最初の15秒で何を作ったか言う

2

データと分析は根拠として見せる

3

UIやプロトタイプを大きく見せる

4

最後に改善案と質問を出す

COLAB PRACTICE

Pythonで試してみる

Google Colabに貼り付けて動かせる練習コードです。まずはそのまま動かし、次に自分たちの列名に置き換えます。

COLAB / PYTHONコピーして試せる
# 3分ピッチの型
0:00-0:20  私たちが気づいたこと
0:20-0:50  作りたいプロダクト
0:50-1:30  集めたデータと分析
1:30-2:20  UI・プロトタイプの紹介
2:20-2:50  工夫した点・改善したい点
2:50-3:00  企業の方に聞きたいこと

OPEN DATA MISSION

オープンデータ演習

IrisやSSDSEの分析結果を使って、1分のミニピッチを作る。「何を見つけたか」「誰に役立つか」「次に何をしたいか」を必ず入れる。

YARUKI LABで使うなら

例：作ったアプリやダッシュボードを、企業の方が“もっと見たい”と思うストーリーにする。

オープンデータの入口

  • SSDSE（教育用標準データセット）：都道府県・市区町村などの公的統計をCSV/Excelで利用できます。
  • e-Stat：政府統計の総合窓口。CSVダウンロードやAPI利用ができます。
  • scikit-learn datasets：Irisなどの練習用データセットをColabですぐ使えます。
